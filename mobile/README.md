# rag-dog Mobile App

React Native (Expo SDK 56) client for rag-dog. Provides Google Sign-In, document upload, and chat against your local RAG backend.

## Prerequisites

- Node.js 20+, pnpm
- Xcode (iOS) or Android Studio (Android)
- A running rag-dog backend (`backend/`) accessible from your device on the LAN

## Configuration — `app.json` extra block

Edit the `extra` section of `app.json` before building:

```json
"extra": {
  "apiBase": "http://192.168.x.x:8000",
  "googleWebClientId": "<web-client-id>.apps.googleusercontent.com",
  "googleIosClientId": "<ios-client-id>.apps.googleusercontent.com"
}
```

| Field | Notes |
|---|---|
| `apiBase` | URL of the FastAPI backend. Use your machine's **LAN IP** (e.g. `192.168.1.10`), **not** `localhost`, when running on a physical device. |
| `googleWebClientId` | OAuth 2.0 Web client ID from Google Cloud Console. |
| `googleIosClientId` | OAuth 2.0 iOS client ID from Google Cloud Console. |

> **Important:** `googleWebClientId` must also be added to the backend's `GOOGLE_CLIENT_IDS` environment variable (comma-separated list) so the backend will accept tokens issued by it.

## Google OAuth setup

1. Open [Google Cloud Console](https://console.cloud.google.com/) → APIs & Services → Credentials.
2. Create an **OAuth 2.0 Web client** and note the client ID (`googleWebClientId`).
3. Create an **OAuth 2.0 iOS client** (bundle ID must match your app, e.g. `com.yourorg.ragdog`) and note the client ID (`googleIosClientId`).
4. Copy the **reversed iOS client ID** (e.g. `com.googleusercontent.apps.<id>`) and replace the `iosUrlScheme` placeholder in the `@react-native-google-signin/google-signin` plugin config in `app.json`:

```json
["@react-native-google-signin/google-signin", {
  "iosUrlScheme": "com.googleusercontent.apps.<YOUR-IOS-CLIENT-ID>"
}]
```

## Dev / EAS build (required)

The `@react-native-google-signin/google-signin` native module does **not** work inside Expo Go. You must use a dev build or a full build:

```bash
# iOS simulator / device (builds a local dev client)
npx expo run:ios

# Android emulator / device
npx expo run:android

# Or trigger an EAS development build (see eas.json)
eas build --profile development --platform ios
eas build --profile development --platform android
```

## Install dependencies

```bash
cd mobile
pnpm install
```

## User allowlist

The signed-in user's Google email must be on the backend allowlist (bootstrap admin or added by an admin) before they can access the API. Check the backend's admin endpoints to add users.

## TypeScript & generated types

`expo-env.d.ts` is **gitignored** — it is auto-generated by Expo and must exist before `tsc` will pass. On a fresh checkout or in CI, generate it by running:

```bash
npx expo start   # then Ctrl-C once the bundler starts
# or
npx expo customize
```

After that, `pnpm tsc --noEmit` and `pnpm lint` will pass cleanly.
